Digitmaps

You supply two values for each digitmap you set up: a dial plan for a valid phone number type, and the SIP address to which phone numbers of that type should be routed. Digitmaps provide flexibility for installations that use different SIP addresses or servers for calls made to internal and external numbers, or for local and long distance numbers. You can also identify multiple acceptable phone number lengths and formats to automate dialing with this setting.

To add a digitmap, click the add icon  .  A popup window prompts you for the dial plan and SIP address. After you complete all of your edits for the device or device group, click Save.

Dial plans

A dial plan establishes the expected length and format of a typical, valid telephone number. When an end user dials a number that has the same length and format as a predefined dial plan, the device recognizes that dialing pattern and routes it to the appropriate SIP address automatically when it is complete.

SIP addresses

The address portion of a Digitmap identifies a SIP Proxy server, SIP Redirect server, SIP gateway, or SIP user agent.

Timer

T

Indicates that an additional time out period, totaling four seconds, should take place before automatic dialing begins.

Only one timer can be included per dial plan, and it must be the last character in the dial plan.

The # character should not be used in dial plans. Pingtel phones reserves # for users to indicate that dialing is complete, like the “send” button on a mobile phone.

Variables in SIP addresses

You have the option to supply the string {digits} or {vdigits} as a part of any SIP address.  

  • {digits} represents the complete string of all dialed digits.

  • {vdigits} represents only dialed digits left unspecified in the dial plan (that is, represented by wildcards).

When a user dials a number that matches the dial plan, the dialed digits are substituted for this placeholder string to establish the SIP address.

The host name is optional: if you specify {digits}@ or {vdigits}@ as the SIP address, the address of your Location server is supplied as the host name.
